## 運用ルール
- 1件は短く固定する
- 再発防止に効くものだけ残す

## ログ
- 2026-03-29 Emote UI のレイアウト崩れ（解決済み）
  - 現象：UI Toolkit 化したエモートUIが非表示または縦並びになった
  - 原因：UXML と USS と共通スタイルの組み合わせで、狙った横並び配置が安定しなかった
  - 対応：共通スタイル依存を外し、`EmoteUIController` で最小レイアウトを直接生成する形へ切り替えた
  - 再発防止：プレイ中の小さな補助UIは、まず最小の単独構成で成立させてから装飾を足す
  - 学び：表示崩れの切り分けでは、資産依存を減らした最小実装に一度戻すほうが早い
- 2026-03-04 Relay依存の版不整合（解決済み）
  - 現象：Relay 関連パッケージ導入時に依存解決エラーが発生した
  - 原因：非推奨化された Relay 単体パッケージ前提で依存を扱った
  - 対応：Unity Editor 上で依存を手動調整した
  - 再発防止：パッケージ導入は Editor 主導で現行推奨構成を確認してから行う
  - 学び：SDK 統合方針を先に確定してから依存追加する
- 2026-03-28 UI Toolkit USS 互換差分（解決済み）
  - 現象：`gap` と `last-child` で StyleSheet 警告が出た
  - 原因：この環境の UI Toolkit が一部の CSS 記法に未対応だった
  - 対応：`margin` と専用クラスへ置き換えた
  - 再発防止：USS は対応プロパティとセレクタだけを使う
  - 学び：Web CSS の感覚で書かず、Unity USS の互換範囲に寄せる
- 2026-03-28 Lobby 一覧更新の 429（解決済み）
  - 現象：RoomBrowser の一覧更新で `Too Many Requests` が発生した
  - 原因：一覧取得の頻度が高く、Lobby API の制限に当たった
  - 対応：自動更新間隔を緩和し、429 時にクールダウンを入れた
  - 再発防止：定期更新は保守的にし、失敗時は即リトライしない
  - 学び：一覧系 API は平常時の更新頻度より制限時の振る舞いが重要
